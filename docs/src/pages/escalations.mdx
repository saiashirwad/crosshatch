# Escalations

Escalations are manual payment approvals. They are useful when an app should ask the user to approve a specific paid
action instead of relying on an autonomous allowance.

## Where Escalations Fit

Allowances define what an app is permitted to spend autonomously. Escalations are explicit approvals for individual paid
actions.

::::steps

### Merchant Prices A Resource

The merchant returns a payment requirement for the resource the app requested.

### User Approves The Escalation

Crosshatch presents the payment to the user for approval.

### Payload Is Signed

Crosshatch signs an x402 payment payload from the user's wallet context.

### Resource Is Delivered

The merchant verifies the payload and serves the paid resource.

::::

## User Experience

Escalations keep the user in the loop. Use allowances when the app needs repeated or autonomous payments; use
escalations when the product experience should ask before spending.
