# Client Quickstart

Crosshatch lets clients call paid x402 services through normal HTTP clients.
When a service returns `402 Payment Required`, Crosshatch creates the payment
payload and retries the request with the payment attached.

## Stablecoin Setup

To pay for x402 resources, your client needs a mnemonic from which funded
signers can be derived.

The mnemonic is the secret from which Crosshatch derives signing capability
across supported networks. Any chain-specific signer used to authorize a payment
ultimately comes from this phrase, so keep it private and make sure at least one
derived signer has the stablecoin required by the resource you want to fetch.

## Call a Paid AI Model

Provide `ChxHttp.layerClient` with a `Payer` layer, then use the client
normally. For OpenAI-compatible x402 services like Blockrun, this works directly
with Effect AI.

```ts
import { OpenAiClient, OpenAiLanguageModel } from "@effect/ai-openai-compat"
import { ChxHttp } from "crosshatch"
import { Console, Effect, Layer } from "effect"
import { LanguageModel } from "effect/unstable/ai"

import { PayerLive } from "./PayerLive.ts"

const BlockrunLive = OpenAiLanguageModel.layer({
  model: "deepseek/deepseek-chat",
}).pipe(
  Layer.provide(OpenAiClient.layer({ apiUrl: "https://blockrun.ai/api/v1" })),
)

LanguageModel.generateText({
  prompt: "Hello from Crosshatch.",
}).pipe(
  Effect.tap(({ text }) => Console.log(text)),
  Effect.provide(
    BlockrunLive.pipe(
      Layer.provide(ChxHttp.layerClient.pipe(Layer.provide(PayerLive))),
    ),
  ),
  Effect.runFork,
)
```

## Fetch a Paid Resource

The same layer also works with ordinary Effect HTTP requests. If the first
response is `402`, Crosshatch reads the payment requirement, asks `PayerLive`
for a payload, and retries once with the payment header attached.

```ts
import { ChxHttp } from "crosshatch"
import { Effect, Layer } from "effect"
import { HttpClient, HttpClientRequest } from "effect/unstable/http"

import { PayerLive } from "./PayerLive.ts"

const program = Effect.gen(function* () {
  const client = yield* HttpClient.HttpClient
  const response = yield* HttpClientRequest.get(
    "https://example.com/paid",
  ).pipe(client.execute)
  return yield* response.text
}).pipe(Effect.provide(ChxHttp.layerClient.pipe(Layer.provide(PayerLive))))
```

`PayerLive` wires the payer to supported assets and signers. The headless
example includes EIP-155 and Solana support:

```ts
import { Accept, KnownAssets, Mnemonic, Payer } from "crosshatch"
import { Eip155Signer, Erc3009Scheme, Permit2Scheme } from "crosshatch/Eip155"
import { SolanaScheme, SolanaSigner, SolanaState } from "crosshatch/Solana"
import { Config, Layer } from "effect"

export const PayerLive = Payer.layer.pipe(
  Layer.provide(
    Accept.layer(KnownAssets.Usd).pipe(
      Layer.provide(
        Layer.mergeAll(
          Layer.mergeAll(Erc3009Scheme.layer, Permit2Scheme.layer).pipe(
            Layer.provide(Eip155Signer.layerMnemonic),
          ),
          SolanaScheme.layer.pipe(
            Layer.provide([
              SolanaSigner.layerMnemonic,
              Config.string("SOLANA_RPC_URL").pipe(
                Config.map(SolanaState.layer),
                Layer.unwrap,
              ),
            ]),
          ),
        ).pipe(Layer.provide(Mnemonic.layerEnv)),
      ),
    ),
  ),
)
```

## Delegate Payment Creation

When another agent should handle approval and signing, delegate payment creation
through a `Bridge`.

```ts
import { Bridge } from "crosshatch"
import { Effect } from "effect"

const program = Effect.gen(function* () {
  const { payload } = yield* Bridge.propose({ required }) // [!code focus]
})
```
