# Merchant Quickstart

Use this guide when your service needs to charge clients for access to a
resource.

Crosshatch helps merchants declare payment requirements, return 402 responses,
read payment payloads, and settle paid requests.

## Merchant Treasury

Consider using [Splits Teams](https://teams.splits.org/) to manage the treasury
or shared pot of funds that receives revenue.

## Protect a Route

Declare what the resource costs and return a 402 response when the request does
not include a payment payload. This example uses HTTP, but Crosshatch is not
limited to HTTP; route protection is just one integration path.

```ts
import {
  Http402,
  Payload,
  Required,
  Requirements,
  KnownAssets,
} from "crosshatch"
import { Effect } from "effect"

const route = Effect.gen(function* () {
  const payload = yield* Payload.Payload

  if (!payload) {
    const required = yield* Required.make`Access to the paid resource.`.pipe(
      Required.accept(
        Requirements.asset(KnownAssets.USDC, {
          amount: 0.01,
          recipients: {
            eip155: { 8453: "0x..." },
          },
        }),
      ),
    )

    return yield* Http402.require({ required })
  }

  return payload
})
```

## Settle a Payment

After receiving a payment payload, settle it before returning the paid resource.

```ts
import { Facilitator } from "crosshatch"

yield * Facilitator.settle({ payload })
```

## Effect HTTP

Crosshatch includes a small Effect HTTP integration for the common merchant
flow:

- Parse an incoming payment payload from the request headers.
- Make the parsed payload available as an Effect service.
- Return an x402 payment-required response when no payload was supplied.
- Settle the supplied payload before returning the paid resource.

Use this integration when your merchant service is built with
`effect/unstable/http`.

### Install The Middleware

`Http402.layerMiddleware()` is an `HttpRouter` middleware. It reads
`payment-signature`, decodes the base64-encoded x402 payload, and provides it to
the route as `Payload.Payload`.

```ts
import { Http402 } from "crosshatch"
import { Layer } from "effect"
import { HttpRouter } from "effect/unstable/http"

const app = HttpRouter.empty.pipe(
  Layer.provide(Http402.layerMiddleware()),
  HttpRouter.toHttpEffect,
)
```

If the header is absent or cannot be decoded, the provided payload is
`undefined`. Your route can then decide whether to return a payment requirement
or continue with unpaid behavior.

### Protect An Effect HTTP Route

Inside the route, read the parsed payload service. If there is no payload, build
a `Required` value and return `Http402.require({ required })`.

```ts
import {
  Http402,
  KnownAssets,
  Payload,
  Required,
  Requirements,
} from "crosshatch"
import { Eip155Address } from "crosshatch/Eip155"
import { Config, Effect } from "effect"
import { HttpServerResponse } from "effect/unstable/http"

const paidRoute = Effect.gen(function* () {
  const payload = yield* Payload.Payload

  if (!payload) {
    const required = yield* Required.make`Access to the paid resource.`.pipe(
      Required.accept(
        Requirements.asset(KnownAssets.USDC, {
          amount: 0.01,
          recipients: {
            eip155: {
              8453: yield* Config.schema(
                Eip155Address.Eip155Address,
                "PAY_TO_EIP155",
              ),
            },
          },
        }),
      ),
    )

    return yield* Http402.require({ required })
  }

  return HttpServerResponse.text("The paid resource.")
})
```

`Http402.require` returns an empty HTTP response with the encoded requirement in
the `payment-required` header. If a Crosshatch trace id is available, it also
includes `x-crosshatch-trace-id`.

### Settle Before Delivery

Most merchant routes should settle the payload before returning the paid
resource.

```ts
import { Facilitator, Http402, Payload } from "crosshatch"
import { Effect } from "effect"
import { HttpServerResponse } from "effect/unstable/http"

const paidRoute = Effect.gen(function* () {
  const payload = yield* Payload.Payload

  if (!payload) {
    return yield* Http402.require({ required })
  }

  const settlement = yield* Facilitator.settle({ payload })

  return HttpServerResponse.text("The paid resource.").pipe(
    Http402.addResponseHeader(settlement),
  )
})
```

`Facilitator.settle({ payload })` uses the configured facilitator client. It
fails the route if the facilitator cannot settle the payment or returns an
unsuccessful settlement result.

### Non-blocking Settlement

To avoid blocking the resource delivery, you can fork a scope-bound fiber from
the settlement effect.

```ts
import { Facilitator, Http402, Payload } from "crosshatch"
import { Effect } from "effect"
import { HttpServerResponse } from "effect/unstable/http"

const paidRoute = Effect.gen(function* () {
  const payload = yield* Payload.Payload

  if (!payload) {
    return yield* Http402.require({ required })
  }

  yield* Facilitator.settle({ payload }).pipe(Effect.forkScoped) // [!code focus]

  return HttpServerResponse.text("The paid resource.")
})
```

### Browser CORS

Browser clients need access to the `payment-required` response header so they
can read the invoice and retry with a payment payload. Crosshatch exports
`Http402.EXPOSED_HEADERS` for the headers used by the HTTP integration.

```ts
import { Http402 } from "crosshatch"
import { Layer } from "effect"
import { HttpRouter } from "effect/unstable/http"

const app = router.pipe(
  Layer.provide([
    HttpRouter.cors({
      allowedHeaders: ["*"],
      allowedMethods: ["*"],
      allowedOrigins: ["*"],
      exposedHeaders: Http402.EXPOSED_HEADERS,
    }),
    Http402.layerMiddleware(),
  ]),
  HttpRouter.toHttpEffect,
)
```

Tighten `allowedOrigins`, `allowedMethods`, and `allowedHeaders` for production
services.
