# Effect HTTP

Crosshatch includes a small Effect HTTP integration for the common merchant flow:

- Parse an incoming payment payload from the request headers.
- Make the parsed payload available as an Effect service.
- Return an x402 payment-required response when no payload was supplied.
- Settle the supplied payload before returning the paid resource.

Use this page when your merchant service is built with `effect/unstable/http`.

## Install The Middleware

`Http402Middleware.layer` is an `HttpRouter` middleware. It reads `payment-signature` or `x-payment`, decodes the
base64-encoded x402 payload, and provides it to the route as `Http402Payload.Http402Payload`.

```ts
import { Http402Middleware } from "crosshatch/Http402"
import { Layer } from "effect"
import { HttpRouter } from "effect/unstable/http"

const app = HttpRouter.empty.pipe(Layer.provide(Http402Middleware.layer), HttpRouter.toHttpEffect)
```

If the header is absent or cannot be decoded, the provided payload is `undefined`. Your route can then decide whether to
return a payment requirement or continue with unpaid behavior.

## Protect A Route

Inside the route, read the parsed payload service. If there is no payload, build a `Required` value and return
`Http402Payload.require({ required })`.

```ts
import { Required, Requirements } from "crosshatch"
import { EvmAddress } from "crosshatch/Evm"
import { Http402Payload } from "crosshatch/Http402"
import { USDC } from "crosshatch/KnownAsset"
import { Effect } from "effect"
import { HttpServerResponse } from "effect/unstable/http"

const paidRoute = Effect.gen(function* () {
  const payload = yield* Http402Payload.Http402Payload

  if (!payload) {
    const required = yield* Required.builder({
      url: "https://merchant.example/paid",
    }).pipe(
      Required.accept(
        Requirements.group(USDC, {
          amount: 0.01,
          recipients: {
            eip155: {
              8453: yield* EvmAddress.config("PAY_TO_EVM"),
            },
          },
        }),
      ),
    )`Access to the paid resource.`

    return yield* Http402Payload.require({ required })
  }

  return HttpServerResponse.text("The paid resource.")
})
```

`Http402Payload.require` returns an empty HTTP response with the encoded requirement in the `payment-required` header.
If a Crosshatch trace id is available, it also includes `x-crosshatch-trace-id`.

## Settle Before Delivery

Most merchant routes should settle the payload before returning the paid resource.

```ts
import { settle } from "crosshatch"
import { Http402Payload } from "crosshatch/Http402"
import { Effect } from "effect"
import { HttpServerResponse } from "effect/unstable/http"

const paidRoute = Effect.gen(function* () {
  const payload = yield* Http402Payload.Http402Payload

  if (!payload) {
    return yield* Http402Payload.require({ required })
  }

  yield* settle({ payload })

  return HttpServerResponse.text("The paid resource.")
})
```

`settle({ payload })` uses the configured facilitator client. It fails the route if the facilitator cannot settle the
payment or returns an unsuccessful settlement result.

## Non-blocking Settlement

To avoid blocking the resource delivery, you can fork a scope-bound fiber from the settlement effect.

```ts
import { settle } from "crosshatch"
import { Http402Payload } from "crosshatch/Http402"
import { Effect } from "effect"
import { HttpServerResponse } from "effect/unstable/http"

const paidRoute = Effect.gen(function* () {
  const payload = yield* Http402Payload.Http402Payload

  if (!payload) {
    return yield* Http402Payload.require({ required })
  }

  yield* settle({ payload }).pipe(Effect.forkScoped) // [!code focus]

  return HttpServerResponse.text("The paid resource.")
})
```

## Browser CORS

Browser clients need access to the `payment-required` response header so they can read the invoice and retry with a
payment payload. Crosshatch exports `EXPOSED_HEADERS` for the headers used by the HTTP integration.

```ts
import { EXPOSED_HEADERS, Http402Middleware } from "crosshatch/Http402"
import { Layer } from "effect"
import { HttpRouter } from "effect/unstable/http"

const app = router.pipe(
  Layer.provide([
    HttpRouter.cors({
      allowedHeaders: ["*"],
      allowedMethods: ["*"],
      allowedOrigins: ["*"],
      exposedHeaders: EXPOSED_HEADERS,
    }),
    Http402Middleware.layer,
  ]),
  HttpRouter.toHttpEffect,
)
```

Tighten `allowedOrigins`, `allowedMethods`, and `allowedHeaders` for production services.
