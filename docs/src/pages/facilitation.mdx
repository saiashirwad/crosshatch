# Facilitation

Settlement and verification are performed through a facilitator service. The
`Facilitator` module exposes `verify({ payload })` and `settle({ payload })`.
Both operations require a `Facilitator` service in the Effect context.

Use `Facilitator.layer()` for the public Crosshatch facilitator, or
`Facilitator.layer({ baseUrl })` for another deployment.

## Crosshatch Facilitator

`layer()` uses the hosted facilitator at `https://cirque.sh` by default.

```ts
import { Facilitator } from "crosshatch"
import { Layer } from "effect"
import { FetchHttpClient } from "effect/unstable/http"

const FacilitatorLive = Facilitator.layer().pipe(
  Layer.provide(FetchHttpClient.layer),
)
```

Provide an Effect HTTP client such as `FetchHttpClient.layer` when the runtime
does not already supply one.

## Providing a Facilitator

Provide `Facilitator.layer` when your program should use a different facilitator
URL.

```ts
import { Facilitator } from "crosshatch"
import { Effect, Layer } from "effect"
import { FetchHttpClient } from "effect/unstable/http"

const Program = Effect.gen(function* () {
  // Your merchant routes or settlement effects live here.
})

Program.pipe(
  Effect.provide(
    Facilitator.layer({
      baseUrl: "https://facilitator.example.com",
    }).pipe(Layer.provide(FetchHttpClient.layer)),
  ),
)
```

Any `Facilitator.verify({ payload })` or `Facilitator.settle({ payload })` call
inside the provided scope uses this facilitator.

## Merchant Settlement

Most merchant servers provide the facilitator client at the application layer so
every paid route settles against the same service.

```ts
import { Facilitator, Payload, Required } from "crosshatch"
import { Effect, Layer } from "effect"
import { FetchHttpClient } from "effect/unstable/http"

declare const required: Required.Required

const route = Effect.gen(function* () {
  const payload = yield* Payload.Payload
  if (!Payload.isAcceptable(required.accepts, payload)) {
    return required
  }
  const settlement = yield* Facilitator.settle({ payload })
  // ...
}).pipe(
  // [!code focus:start]
  Effect.provide(
    Facilitator.layer({
      baseUrl: "https://facilitator.example.com",
    }).pipe(Layer.provide(FetchHttpClient.layer)),
  ),
  // [!code focus:end]
)
```

## When to Override

Use a custom facilitator when you operate your own settlement service, route
traffic through an environment-specific deployment, or need to test settlement
against a local facilitator.

```ts
import { Facilitator } from "crosshatch"

Facilitator.layer({
  baseUrl: "http://127.0.0.1:4647",
})
```

The service must implement the `FacilitatorApi` exported from
`crosshatch/FacilitatorApi`. Its mounted endpoints are
`POST /facilitator/verify`, `POST /facilitator/settle`, and
`GET /facilitator/supported`. The supported-network schema accepts both CAIP-2
`ChainId` values and legacy network strings.

## Implementing a Facilitator

The Effect HTTP API definition is exported from its own package subpath:

```ts
import { FacilitatorApi, FacilitatorApiGroup } from "crosshatch/FacilitatorApi"
```

Use `FacilitatorApi` when implementing or mounting a compatible facilitator
server.

## Mock Settlement

Use the development facilitator to exercise payment flows without moving funds.
It accepts every verification and returns a placeholder successful settlement;
it does not verify signatures or broadcast transactions.

Start it on a known port:

```sh
crosshatch dev --port 4647
```

Then reserve `CROSSHATCH_DEV_PORT` in the application process to make
`Facilitator.layer()` use it:

```sh
CROSSHATCH_DEV_PORT=4647 your-dev-command
```

`CROSSHATCH_DEV_HOST` is also reserved and defaults to `localhost`. When
`CROSSHATCH_DEV_PORT` is set, these variables take precedence over `baseUrl` and
the hosted facilitator. Match `--host` and `CROSSHATCH_DEV_HOST` when the server
is not local.
