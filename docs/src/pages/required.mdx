# Payment Required

`Required` is Crosshatch's builder for x402 payment-required responses.

You can think of this as the "invoice" format of x402. It is the description a
server returns when a client asks for a paid resource without providing payment.
The client reads the invoice, selects one of the accepted payment requirements,
creates a [payment payload](/payload), and retries the request with that payment
attached.

## Building a Required

Start with `Required.make`, then pipe builder operations into it.
`Required.accept` sets the acceptable payment requirements, replacing any
previous entries. `Required.extend` adds x402 extension payloads.

```ts
import { Config, Effect } from "effect"
import { Required, Requirements, KnownAssets } from "crosshatch"
import { Eip155Address } from "crosshatch/Eip155"
import { PaymentId } from "crosshatch/Extensions"

const makeRequired = Effect.gen(function* () {
  const recipient = yield* Config.schema(
    Eip155Address.Eip155Address,
    "PAY_TO_EIP155",
  )

  return yield* Required.make`
  | Access to the paid resource.
  | This invoice is for a single request to /paid.
  `.pipe(
    Required.extend(PaymentId.FromMerchant, {
      required: true,
      id: PaymentId.random(),
    }),
    Required.accept(
      Requirements.logical(KnownAssets.Usd.USDC, {
        amount: 0.01,
        recipients: { eip155: { 8453: recipient } },
      }),
    ),
  )
})
```

The description is passed first. It should be written for the client or wallet
that may show the payment request to a user.

## Requirements Groups

`Requirements.logical` turns one logical asset's namespace map into acceptable
payment options. For example, `USDC` may contain physical assets for multiple
EIP-155 chains. The `recipients` object chooses which deployments this invoice
accepts and where payment should be sent on each chain.

```ts
import { KnownAssets, Requirements } from "crosshatch"
import { Eip155Address } from "crosshatch/Eip155"

declare const payToOnBase: typeof Eip155Address.Eip155Address.Type

Requirements.logical(KnownAssets.Usd.USDC, {
  amount: 0.01,
  recipients: {
    eip155: {
      8453: payToOnBase,
    },
  },
})
```

The USD amount is converted into the asset's atomic units using the selected
deployment's decimals.

Use `Requirements.denomination` when you want to accept every configured asset
in a denomination, such as any asset in `KnownAssets.Usd`, with one recipient
configuration.
