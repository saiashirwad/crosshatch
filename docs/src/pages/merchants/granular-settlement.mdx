# Granular Settlement

Granular settlement prices each resource independently.

## When To Use It

Use this strategy when every request maps cleanly to a price: one API response, one generated asset, one media segment,
or one tool call.

## Shape

::::steps

### Price The Resource

The merchant returns an x402 requirement for the requested unit.

### Verify The Payload

The client retries with a signed payment payload.

### Serve The Resource

The merchant serves the paid resource after verification succeeds.

::::

Granular settlement is easiest to reason about because the payment boundary and resource boundary are the same.
